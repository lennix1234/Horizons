About Us

We are 'Bob the Builder', a team of passionate mechanical and design engineers with an innovative product.
Follow our journey and be part of the Stop and Lock revolution!

  • October 2019

    The team's humble beginnings

    It was a rainy October afternoon in London. Our team of young engineers had just formed and we were full of the essential energy and motivation to come up with amazing ideas. Our motto and approach to problems soon became: 'Can we fix it? Yes, we can!'

  • November 2019

    Eureka! The solution to was so clear

    The idea for Stop and Lock sparked from a simple, yet important issue: locking doors in student and shared housing. There are no locks on internal doors to safeguard your privacy and security. Sharers do face situations where they need temporary locking of rooms, cupboards and closets.

  • December 2019

    Refining down the product features

    A period of restless market research and testing began. We had to create THE perfect product and identifying its features was key to our success. Thin doors, double doors, cupboards... we had to start from somewhere and agile planning gave us the answer with the MoSCow prioritazion framework. We had all the ingredients ready to change the world!

  • December 2019 -

    The prototyping begins

    Prototype after prototype we tested ideas, discarded a few, refined others. It was the peak of the creativity and the also the start of the fun. Bob the Builder knew how to fix it and it was just a matter of time before Stop and Lock in its final form would become a reality.

Additional Information

This section will go over the tools used in the creation of this website. It will go through the steps taken to setup the domain and will also go through what makes a good website.

Design tools used

  1. The website was built around a Bootstrap template, which provided a free and open-source CSS framework directed at responsive, mobile-first front-end web development.
  2. The approach of "cowboy" coding was used in order to simplify the task and make use of open-source interface elements designed by others. Elements were taken from other Bootstrap templates, Github forums and CSS and JS code snippets from Bootsnipp.
  3. Using graphic design tools to generate images, change the colours and the sizes to fit the web page.
  4. Github was used for version control and hosting of the web page.
  5. Atom was used as teh chosen web development environment to edit the source code.

Domain setup

Creating a custom domain name

In order to register a domain, the simplest way is to purchase a private domain with a registrar and web hosting company. In this case GoDaddy was used to purchase lorenzobergamaschi.com. The next step was to access DNS management and point the custom domain to GitHub’s server over HTTPS. The IP addresses of GitHub's server were added. A file named "CNAME", containing the custom domain name was created in the repository of the website in the root of the directory. Once the domain certificate was issued and the website allowed to be served through HTTPS, the website was online.

Websites

This section will go over what in my opinion makes a good website. It also will describe accessibility, GDPR, privacy, marketing and future steps.

What makes a good website?

Quick and easy steps to make an attractive page

  1. It needs to be responsive! Nothing is more key than being able to render web pages well on a variety of devices and window or screen sizes.
  2. A Great balance between User Interface (UI) and User Experience (UX): a website which is easy, fun and attractive to navigate.
  3. The quality of the content is of crucial importance. It needs to be engaging and of interest to the target user.
  4. The webpage has to be light and fast to load. No assumptions can be made on the quality of the internet connections used.
  5. Use of pretty pictures and visuals can really help UI.
  6. Avoid too many words on the page. Focus on simplicity, blank space and breathing space.

Disability-friendly considerations

What if someone is visually impaired or colorblind?

  1. Making use of alt tags for images is essential to describe the content for visually impaired individiduals.
  2. Any abbreviations contain periods to make screen readers pronounce it correctly. Eg. C.A.D.
  3. Buttons all have a description to allow screen readers to describe the content of the link.
  4. Colour pairing was chosen to allow colour blind users to comfortably view the website content. Black text on white background for text heavy sections was used.
  5. Buttons and links are purposely made large to make them easy to click.
  6. No videos were included in this website, however subtitles would be a must.

GDPR considerations?

Ensuring privacy and safety in an age dominated by data

  1. This website includes an explicit request of consent to obtain personal data from users.
  2. The data is stored in a secure manner and the best security measures will be employed.
  3. There is no content included that would not be suitable for children.
  4. Systems are in place to detect data breaches and communicate them to affected parties.

Marketing the website

How to increase visibility and website traffic?

  1. Starting by accomplishing a good Search Engine Optimization (SEO) to increase the visibility.
  2. Search Engine Marketing (SEM) is also key to promote the website through search engines.
  3. Use social media pages to post the link to the website. Eg. Facebook, Twitter, Instagram...
  4. Improve the website content: include engaging material that users would share.
  5. Make use of online forums to promote the website and its content.
  6. Using flyers and promoting the product at fairs would increase website views by providing QR codes to scan.

With additional time I would...

  1. Improve the UI by adding more links, more website pages and a better navigation.
  2. Create a product marketing theme. Eg. Colour coding, fonts, logos, stock images...
  3. Add an interative GUI to play around with the product.
  4. Set up a database for incoming contact messages and an automatic reply.
  5. Implement more security measures to avoid breaches of data.
  6. Add an e-commerce platform to sell the products.
  7. Enable online secure payment using major credit card circuits and Paypal.
  8. In general, make it prettier!!

First pretotype

The first iteration of concept 1

By bending sheet metal, we managed to test the functionalities of applying normal force to the ground by making use of the door. The idea is that the moment generated by the act of opening the door, would generate an increasing normal force.

  • Date: November 2019
  • Purpose: Mechanism testing

Second Pretotype

A brand new mechanism concept

By laser cutting MDF wood, and using nuts and a threaded rod, we managed to apply a normal force. This was done by using the bottom of the door as a lever and tightening the lead screw.

  • Date: December 2019
  • Purpose: Mechanism testing

Refined concept 1

A rigid structure and a strap.

Using laser cut MDF and a ratchet strap, we attempted to apply a normal force on the door's lower surface. More work was needed to refine the current design.

  • Date: January 2020
  • Purpose: Refined mechanism testing

Refined concept 2

Using stronger materials

To resist the great force applied to the door, laser cut 2mm stainless steel plate was used. Making use of bolts and tightening them, we applied a normal force making use of friction to stop the door.

  • Date: January 2020
  • Purpose: Refined Mechanism testing
